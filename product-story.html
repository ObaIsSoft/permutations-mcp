🔀 PERMUTATIONS MCP

A Mathematical Design Engine for the AI Era

The Problem: Template Entropy

Every AI-generated website looks identical. Inter font. Blue-purple gradients. Rounded cards. Glassmorphism overlays. The same Figma templates recycled through ChatGPT, Claude, and Cursor. We've reached peak homogeneity in AI-assisted design.

This isn't just boring—it's brand death. When every startup, every portfolio, every dashboard looks the same, differentiation becomes impossible. Users can't distinguish products. Companies can't establish identity. The web becomes a sea of sameness.

The Gap: Intent-to-Design Translation

Current AI coding tools (Cursor, Claude Code, Windsurf) are pattern regurgitators. They:

  • Pull from training data biases (Inter is mentioned 10,000x more than Space Grotesk)
  • Default to "safe" choices (blue gradients, rounded corners)
  • Have no concept of contextual appropriateness
  • Generate the same output for "fintech dashboard" and "yoga studio"

What's missing: A constraint layer that forces AI to generate contextually-appropriate, mathematically-unique designs based on actual content and intent—not training data biases.

The Solution: Biological Design DNA

Permutations treats design like evolutionary biology. Just as life adapts to planetary environments (polar bears don't look like tropical fish), we force AI to adapt UI to content environments.

How It Works

  1. Planetary Survey: Analyze content across 5 semantic vectors:
    Information Density Temporal Urgency Emotional Temperature Playfulness Spatial Dependency
  2. Genetic Sequencing: Inject vectors into SHA-256. Same planet, same organism—always.
  3. Epigenetic Override: Brand assets (logos, PDFs) forcibly mutate DNA to match existing identity.
  4. Constraint Injection: MCP server injects topology, color, motion rules directly into AI IDE.

The Logo: The Permutation Symbol

Our identity is the permutation symbol—mathematical notation for rearrangement. It represents:

  • Infinite possibility from finite elements
  • Deterministic output from specific inputs
  • The mathematical beauty of adaptation

Content Traits for This Product

Permutations is itself a technical developer tool. Its design should reflect:

  • High Information Density: Technical documentation, code samples, API references
  • Medium Temporal Urgency: Product landing page—neither dashboard-fast nor blog-slow
  • Clinical/Technical Tone: Developer audience, not consumer emotional appeal
  • Low Playfulness: Serious tool, brutalist aesthetic—not whimsical or cute
  • Medium Spatial Dependency: Some 3D visualization to demonstrate capability, mostly flat UI

Functional Archetype

Type: Technical Documentation + Product Landing Page

Primary Action: Install MCP, understand methodology, view examples

Audience: Developers, AI researchers, design engineers

The Design Must Demonstrate

  1. Dark Theme: Developer tools live in dark mode
  2. High Contrast: WCAG AAA for technical readability
  3. Geometric Typography: Clinical precision, not humanist warmth
  4. Visible DNA: Show the hash, traits, chromosomes—prove it's generated
  5. Working Example: The website itself should use its own generated DNA
  6. Metallic Material: Cold, precise, industrial aesthetic
  7. No Motion: Respect reduced-motion preference (serious = accessible)

Key Sections Needed

  1. Hero/Manifesto: "Life Adapts. Slop Doesn't." The anti-template thesis.
  2. How It Works: The 4-step biological process (Survey → Seed → Epigenetics → Injection)
  3. Live DNA Display: Show actual generated genome, traits, chromosomes
  4. Installation: MCP setup for Cursor/Claude/Windsurf
  5. Examples Gallery: Before/after (generic AI slop vs Permutations DNA)
  6. Technical Architecture: MCP protocol, constraint solver, pattern detection
